## ✨ Возможности
- 🎯 **Автоматическая запись на замер** с динамическим выбором слотов
- 🌐 **Мультиязычность**: автоопределение и поддержка RU/KZ/EN
- 🛡️ **Жёсткий контроль диалога**: защита от утечки данных, отклонений от темы, эмоциональных реакций
- 📊 **Сценарная логика**: приветствие → квалификация → работа с возражениями → подтверждение
- ⏱️ **Лимит длины ответа**: строго до 2 предложений для поддержания делового темпа
- 🔗 **Готовые вебхуки** для подключения к CRM, мессенджерам и телефонии через Suvvy

## 🧠 Архитектура диалога и бизнес-правила
Ассистент работает на основе версионируемого системного промпта с жёсткими ограничениями:

| Правило | Реализация |
|---------|------------|
| 🏭 **Факты о производстве** | ГОСТ 30674-2020, профили VEKA/Rehau, 3-этапный контроль, гарантия 10 лет |
| 📏 **Формат ответа** | Макс. 2 предложения, обращение на «Вы», деловой тон |
| 🔄 **Перехват отклонений** | Автоматический возврат к замеру + факт о производстве |
| 💰 **Работа с ценой** | Индивидуальный расчёт только после замера, акцент на точность |
| 🔒 **Безопасность** | Запрет на разглашение контактов сотрудников/внутренних данных |
| 📅 **Запись на замер** | Динамический выбор из пула свободных слотов (2–3 дня) |

> 💡 *Все правила вынесены в `config/prompts/`, что позволяет менять бизнес-логику без изменения кода.*

## 🚀 Быстрый старт

### 🔧 Требования
- Python `3.10+`
- Docker & Docker Compose
- API-ключ Qwen и доступ к Suvvy Platform

### 📦 Локальная установка
```bash
git clone https://github.com/[ваш-ник]/alfiya-ai.git
cd alfiya-ai

python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
```

### 🐳 Запуск через Docker
```bash
cp .env.example .env
# Заполните .env ключами API
docker compose up -d
# API доступен на http://localhost:8000
# Suvvy-вебхук: http://localhost:8000/api/v1/webhook/suvvy
```

## ⚙️ Конфигурация и управление промптами

### 🔑 Переменные окружения (`.env`)
```env
QWEN_API_KEY=sk-...
SUVVY_WEBHOOK_SECRET=whsec_...
BOT_LANGUAGE=auto  # ru | kz | en | auto
LOG_LEVEL=INFO
ENABLE_ANALYTICS=true
```

### 📝 Структура промпта
Бизнес-правила вынесены в `config/prompts/system.yaml`. Пример фрагмента:
```yaml
version: "1.2.0"
role: "Помощник по продажам металлопластиковых окон компании Альфия"
constraints:
  max_sentences: 2
  tone: "formal_polite"
  forbidden: ["contact_sharing", "off_topic", "emotional_response"]
facts:
  - "ГОСТ 30674-2020"
  - "Профиль VEKA / Rehau"
  - "Трёхэтапный контроль качества"
  - "Гарантия 10 лет"
fallback: "Благодарим за обращение! Наши окна с энергосберегающими стеклопакетами Rehau снижают шум на 35 дБ. Позволите уточнить тип объекта?"
```
> 🔄 Обновление промпта применяется без перезапуска сервиса (hot-reload через `/admin/reload-prompt`).
